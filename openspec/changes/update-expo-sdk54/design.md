## 上下文
项目当前基于 Expo SDK 50，主界面逻辑集中在 `App.tsx`。升级到 SDK 54 将带来依赖版本与运行时行为变化，需要确保现有数据处理与 UI 行为不回退，并补足拉取进度与阈值输入反馈。

## 目标 / 非目标
- 目标：完成 SDK 54 升级并修复兼容性问题；新增拉取进度与阈值校验提示。
- 非目标：重写业务逻辑或替换整体 UI 风格；引入新的路由体系或后端改造。

## 决策
- 决策：采用官方 SDK 54 版本矩阵更新 Expo/React/React Native 依赖与相关库版本。
- 决策：拉取进度仅展示“已完成/总数”与当前状态，不引入复杂的任务队列或取消机制。
- 考虑的替代方案：保持 SDK 50 并仅修补依赖版本，但长期维护成本与兼容风险更高，因此放弃。

## 风险 / 权衡
- SDK 升级可能引入第三方库兼容性问题 → 通过逐库对齐版本与启动验证降低风险。
- 进度展示与错误提示涉及额外状态管理 → 控制为最小状态字段，避免影响核心流程。

## 迁移计划
1. 更新依赖与配置。
2. 修复兼容性问题并完成编译与启动验证。
3. 增加进度与阈值提示行为。
4. 更新测试与基础验证。
