## 上下文
- 需要将 Notebook 的 Python 请求与工时计算逻辑，在 React Native (Expo) 中完全复刻。
- 运行环境为 Windows 11，iOS 预览使用 Expo Go。

## 目标 / 非目标
- 目标：与 Notebook 一致的拉取范围、计算规则、统计结果与异常提示。
- 目标：提供可保存配置、可切换月份的移动端 UI。
- 非目标：后端改造、账号体系或离线批量导入。

## 决策
- 数据拉取：对所选月份与下月各请求一次，合并后仅保留所选月份记录。
- 日期处理：使用 `dayjs` 进行月份加一、日期比较与显示格式化；工时差值以毫秒差换算小时。
- 异常规则：工时 `<= 10.5` 视为不足；打卡时间缺失或无法解析时工时记为 0。
- 平均工时：有效工作日为 0 时显示“异常”，不计算除法。

## 备选方案
- 仅拉取所选月份：被拒绝（与 Notebook 行为不一致）。
- 使用 `date-fns`：可行但当前优先 `dayjs` 的轻量 API。

## 风险 / 权衡
- Authorization 失效或接口返回错误时需要明确提示并避免覆盖已有数据。
- 不同地区设备时区可能影响“今天”判断，需要统一使用本地日期字符串 `YYYY-MM-DD`。

## 迁移计划
- 作为新应用落地，无历史迁移；仅需初始化本地存储键值。
